解析：Derived回调里面InputCallback里的parent_调用到 Base类中的PrintBase函数
```C++
void Derived::InputCallback(char c, void *user_data) {

    if (c == 'a') {
        Derived::Impl* impl = static_cast<Derived::Impl*>(user_data);
        Derived* d = impl->parent_;
        std::cout << "InputCallback" << std::endl;
        d->PrintBase();
    }
}
```

豆包总结：protected 访问的「类属」特性
C++ 的访问控制（public/protected/private）是基于「类的作用域」，而非「对象实例」：
Derived 类的代码（无论成员函数 / 静态函数），都能访问「任意 Derived 对象」中的 Base protected 成员；
关键是「调用代码属于 Derived 类」+「被调用对象是 Derived 类型」，而非「是否是静态函数」或「是否是当前对象」；
你的代码中 d 是 Derived*，且 InputCallback 是 Derived 的静态成员函数，因此调用 d->PrintBase() 完全符合 C++ 访问规则。
